## 瓶颈 1：d4 De_XConv 混合卷积 — ⚠️ 需 MATLAB 协助

### 诊断过程

1. 更新 `calibrate_decoder_v2.c` d4 段为正确的 hybrid conv(H)+tconv(W) 实现
2. 发现 TConv-only SNR = **-5.28 dB**（无论何种 weight indexing 方案）
3. BN 层大幅修正信号 → TConv+BN SNR = **~8.6 dB**
4. 测试了 5+ 种不同的 MATLAB weight layout 假设，均无法突破 -5 dB

### 根因分析

MATLAB `tconv2d_func.m` 对 kernel 做了 `rot90(kernel,2)`（180°旋转），而 `non_gtconv2d_func.m` 做了 `.T` + `rot90(kernel,90)`。两者的 weight 访问模式不同，且 `.mat` 文件的存储 shape 未知。

**关键证据**：
- TConv 输出全部为正（~3.3M），而 golden 为负（~-1M 到 -6M）
- BN 参数 (weight=7565, bias=-1251834, mean=4417920) 在做大幅度极性+幅度修正
- 0.9896 相关性表明信号结构正确但极性/幅度完全错

## 瓶颈 2：DPRNN GRU Q-format — ⚠️ 部分改善 + 需 MATLAB 协助

### 校准结果

创建了 `calibrate_dprnn_gru.c` 对 Intra_RNN 和 Inter_RNN 的 qr1/qr2 进行了暴力搜索：

| Block | 参数 | 原值 | 校准值 | SNR 改善 |
|:--|:--|:--|:--|:--|
| **GDPRNN 0** | Intra qr | -13,-8 | **-14,-6** | -3.29→-2.71 (+0.58 dB) |
| | Inter qr | -13,-8 | **-13,-11** | |
| **GDPRNN 1** | Intra qr | -13,-8 | **-18,-13** | -2.63→-2.08 (+0.55 dB) |
| | Inter qr | -13,-8 | **-16,-13** | |

### 已应用修改

`ulunas_modules.c` 中的 `Intra_RNN_module` 和 `Inter_RNN_module` 已更新为校准后的 QR 值。

### 根因分析

Block 1 的最优 QR 值极为激进（intra_qr1=-18），远超正常范围，说明 **GRU 权重的 Q-format 与实际不匹配**。

文档标注 Q_WGT_GRU=12 (s16f12)，但权重量化分析显示 max_abs=18877，Q14→1.15 更合理。这意味着权重值被低估了 4 倍（2^(14-12)）。

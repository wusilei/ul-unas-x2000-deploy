# MATLAB → C 定点 STFT 转换经验

## MATLAB 定点 Q 格式命名规则

### 格式定义

|          | MATLAB 格式 | C 格式     | 含义                  |
|----------|------------|-----------|-----------------------|
| 有符号    | Q**I**.**F**   | s**W**f**F**  | W = I + F, 补码表示    |
| 无符号    | UQ**I**.**F**  | u**W**f**F**  | W = I + F, 纯幅度      |

### 命名拆解

```
QI.F  = sWfF    其中 W = I + F
UQI.F = uWfF    其中 W = I + F

Q  = Quantized (定点)
UQ = Unsigned Quantized (无符号定点)
I  = 整数位宽 (integer bits)
F  = 小数位宽 (fractional bits)
W  = 总位宽 (total bit width) = I + F

s  = signed (有符号, 补码)
u  = unsigned (无符号)
```

### 常用格式速查表

| MATLAB  | C 格式 | 总位宽 | 整数 | 小数 | 范围 | 精度 | 用途 |
|---------|--------|--------|------|------|------|------|------|
| Q12.20  | s32f20 | 32     | 12   | 20   | [-2048, ~2048] | 2⁻²⁰ ≈ 9.5×10⁻⁷ | 层间激活值 |
| Q1.15   | s16f15 | 16     | 1    | 15   | [-1, ~1] | 2⁻¹⁵ ≈ 3.0×10⁻⁵ | 音频/PCM, GRU隐状态 |
| Q5.11   | s16f11 | 16     | 5    | 11   | [-16, ~16] | 2⁻¹¹ ≈ 4.9×10⁻⁴ | — |
| UQ1.15  | u16f15 | 16     | 1    | 15   | [0, ~2] | 2⁻¹⁵ | sigmoid输出, ERB权重 |
| UQ12.20 | u32f20 | 32     | 12   | 20   | [0, ~4096] | 2⁻²⁰ | 无符号中间量 |

### 转换公式

```
float → sWfF:   x_fixed = round(x_float × 2^F)    // 截断到 W 位有符号范围
float → uWfF:   x_fixed = round(x_float × 2^F)    // 截断到 W 位无符号范围
sWfF → float:   x_float = x_fixed / 2^F
uWfF → float:   x_float = x_fixed / 2^F

Q15 → Q20:      x_q20 = x_q15 << 5                 // 精确, 无舍入
Q20 → Q15:      x_q15 = (x_q20 + 16) >> 5          // 四舍五入
```

### EffBits（有效位宽）

```
EffBits = ⌈log₂(|x_fixed|)⌉    (x_fixed ≠ 0)

含义: 该定点值在 W 位容器中实际占用的二进制位数
EffBits 大 → 信号强, 动态范围利用充分
EffBits ≈ 0 → 信号接近零

注意: EffBits 描述的是动态范围利用率, 与精度(2^-F)无关
      值小不代表"噪声大", 只代表离满量程远
```


## 核心强制约束（必严格遵守）

1. STFT函数定点定标规则
   - 输入参数noisy_audio：定点规格**s16f15**（16位有符号定点数，1位整数位、15位小数位）
   - 输入参数hann_window：定点规格**u16f15**（16位无符号定点数，1位整数位、15位小数位）
   - 输出参数cmp_real、cmp_imag：定点规格**s32f20**（32位有符号定点数，12位整数位、20位小数位严格匹配该定标规则）
2. 精度对齐：所有定点位宽配置、数值定标规则、运算移位逻辑、截断/舍入方式，**100%严格匹配Matlab Fix_point.m及原工程**，不允许自定义修改位宽、移位、定标参数。
3. 函数适配：Matlab中原浮点实现的sigmoid、tanh等函数，禁止直接使用浮点实现，如可用的内置算子，必须手动编写**定点近似实现**，且输入、输出的定点定标规格完全遵循Matlab工程配置。





## 1. 输入对齐检查清单

转换前必须逐项对齐的参数：

| 参数      | MATLAB (STFT_func.m) | 值                | 影响          |
| ------- | -------------------- | ---------------- | ----------- |
| N_fft   | `N_fft`              | 512              | FFT 点数      |
| win_len | `win_len`            | 512              | 帧长          |
| win_inc | `win_inc`            | 256              | 帧移          |
| 窗函数     | `stft_window.mat`    | **直接用 .mat 值 ×32767 取整** | C 代码中用等值 Q15 表 |
| 边界处理    | `flip(x(2:pad+1))`   | **镜像填充** 256 点   | 首尾帧与直通完全不同  |
| 音频归一化   | `audioread`          | **[-1,1] float** | C 侧需 /32768 |

### 最容易漏掉的坑

1. **窗函数值** — 从 `stft_window.mat` 直接导入，`round(value × 32767)` 存为 Q15 表。不可用公式近似，必须逐值对齐
2. **镜像填充** — MATLAB `STFT_func` 对首尾做 256 点镜像填充，C 代码如果直通取帧，第一帧结果完全不匹配
3. **FFT 输出缩放** — 每级蝶形 `>>15` 使输出保持在 Q15，不是 Q15×N

## 2. 定点 Q 格式约定

```
信号路径:  PCM Q15 → 加窗 Q15 → FFT Q15 → Q20 (<<5) → 模型 → Q20 → Q15 (>>5) → IFFT Q15 → OLA → 输出

Q15: 1 位符号 + 15 位小数, 范围 ±1.0, 精度 1/32768 ≈ 3×10⁻⁵
Q20: 1 位符号 + 11 位整数 + 20 位小数, 精度 1/1048576 ≈ 9.5×10⁻⁷
```

### 关键转换

```
float → Q15:  round(x * 32768)         // 饱和到 ±32767
float → Q20:  round(x * 1048576)       // 饱和到 int32 范围
Q15 → float:  x / 32768.0
Q20 → float:  x / 1048576.0
Q15 → Q20:    x << 5                   // ×32, 位精确
Q20 → Q15:    (x + 16) >> 5            // 四舍五入
```

### 中间计算结果用 int32/int64

```
Q15 × Q15 → int64 乘积 → (prod + 16384) >> 15 → Q15
Q15 × Q15 → int64 乘积 → prod >> N     → Q(30-N)
累加器必须是 int64，避免溢出
```

## 3. FFT 定点实现要点

### 旋转因子预计算

```c
// cos(2πk/512) × 32767, k=0..255, Q15
static const int16_t tw_cos[256] = { 32767, 32765, ... };
static const int16_t tw_sin[256] = { 0, 402, 804, ... };
```

### 位反转表

512 项预计算，9-bit 反转，避免运行时计算。

### 蝶形运算

```c
int32_t tr = (real[j2]*wr - imag[j2]*wi) >> 15;  // Q15×Q15 → Q30 → >>15 → Q15
int32_t ti = (imag[j2]*wr + real[j2]*wi) >> 15;
real[j2] = real[j] - tr;   // int32, 不截断
imag[j2] = imag[j] - ti;
real[j]  += tr;
imag[j]  += ti;
```

**关键**：乘法后立即 `>>15`，但加减法在 int32 中执行不截断。这使 DC bin 可以达到 Q15×512 的范围而不溢出。

### 逆 FFT 的共轭对称

```c
// 257 bins → 512 点复数（利用共轭对称性）
for (i = 1; i < 256; i++) {
    r[512-i] =  real[i];    // 共轭对称: F(-k) = F*(k)
    i[512-i] = -imag[i];
}
```

| 误差源 | 贡献 | 说明 |

| 加窗取整 | 次要 | `(s×w+16384)>>15`, ±0.5 LSB |
| 窗函数量化 | 次要 | float → Q15 round |
| 输入量化 | 可忽略 | int16 PCM 本身即 Q15 |

## 5. 验证方法

### 单帧对比

```python
# MATLAB reference (float)
x_norm = audio / 32768.0
xp = mirror_pad(x_norm, 256)
frame = xp[0:512] * window
matlab_fft = np.fft.fft(frame)[:257]

# C output (Q15)
c_fft = parse_c_output()
c_float = c_fft / 32768.0  # Q15 → float (NOT /512!)

snr = 10*log10(sum(|matlab|²) / sum(|matlab - c_float|²))
```

### 归一化确认

C 输出 Q20 的正确归一化: `C_raw / 1048576.0`

```
Q20 → float:  x / 2^20 = x / 1048576.0
Q15 → float:  x / 2^15 = x / 32768.0
Q15 → Q20:    x << 5    (exact, 无取整)
```

验证: `C_Q20[0] / 1048576` 应 ≈ `MATLAB[0]`。

## 6. 代码模板

```c
// 1. 镜像填充
int16_t *xp = malloc((PAD_LEN + n + PAD_LEN) * 2);
for (int i = 0; i < PAD_LEN; i++) xp[i] = x[PAD_LEN - i];   // 前镜像
memcpy(xp + PAD_LEN, x, n * 2);                               // 原始
for (int i = 0; i < PAD_LEN; i++) xp[PAD_LEN+n+i] = x[n-2-i]; // 后镜像

// 2. STFT 主循环
for (int t = 0; t < T; t++) {
    int16_t *frame = xp + t * WIN_INC;
    
    // 加窗: Q15×Q15 → Q15
    int32_t fft_in[512];
    for (int i = 0; i < 512; i++)
        fft_in[i] = ((int64_t)frame[i] * window[i] + 16384) >> 15;
    
    // FFT: Q15 → Q15 (int32 容器)
    int32_t imag[512] = {0};
    fft_q15_512(fft_in, imag, 0);  // in-place
    
    // 输出 257 bins (Q20: Q15 << 5, in int32)
    for (int b = 0; b < 257; b++) {
        real_out[b] = fft_in[b] << 5;   // Q15 → Q20, exact
        imag_out[b] = imag[b] << 5;
    }
}
```

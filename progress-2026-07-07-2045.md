## 本轮修复 (v2 column-major 权重索引 + Qr对齐)

### 1. 权重索引: row-major → column-major (关键修复)

MATLAB `.mat` 文件以 column-major 存储，`extract_weights.m` 使用 `data(:)` 展平时保持 column-major 顺序。但 v2 C 代码使用了 row-major 索引模式，导致所有权重读取被转置。

**已修复的运算符 (ulunas_fp.c):**
- `conv2d_func`: `weight[((co*Cin+ci)*Kh+kh)*Kw+kw]` → `weight[co + Cout*(ci + Cin*(kh + Kh*kw))]`
- `pconv2d_func`: `weight[co*Cin+ci]` → `weight[co + Cout*ci]`
- `gconv2d_func`: `weight[((co*1+0)*Kh+kh)*Kw+kw]` → `weight[co + Cout*(kh + Kh*kw)]`
- `non_gconv2d_func`: 同上 → `weight[co + Cout*(kh + Kh*kw)]`
- `tconv2d_func`: 修复为 `weight[ci + Cin*(co + Cout*((Kh-1-kh) + Kh*(Kw-1-kw)))]`
- `gtconv2d_func`: 修复为 `weight[co + Cout*((Kh-1-kh) + Kh*(Kw-1-kw))]`
- `non_gtconv2d_func`: 修复为 `weight[co + Cout*kw]`
- `bm_fixed`: `weight[i*hi_out+o]` → `weight[i + hi_in*o]`
- `bs_fixed`: 同上 → `weight[i + hi_in*o]`
- `gru_module`: ih_weight `[i*3*nH+j]` → `[i + in_dim*j]`, hh_weight `[i*3*nH+j]` → `[i + nH*j]`
- `ctfa_ta_module`: FC weight `[j*C+c]` → `[j + nHidden*c]`
- `ctfa_fa_module`: FC weight `[i*group+g]` → `[i + in_fc*g]`

**已修复的模块 (ulunas_modules.c):**
- `intra_rnn_module`: FC weight `[i*16+o]` → `[i + 16*o]`
- `inter_rnn_module`: FC weight `[i*16+o]` → `[i + 16*o]`
